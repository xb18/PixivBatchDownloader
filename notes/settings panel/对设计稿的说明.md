## 新版界面解决了哪些问题

### 功能按钮太过分散

在旧版界面里，开始抓取、下载按钮、下载区域、更多按钮是分散在 3 个选项卡里的。默认显示的“抓取”选项卡只会显示开始抓取。

在新版界面里，添加了“首页”作为仪表盘，在这里集中展示所有重要的功能按钮和下载区域，使高频操作不再需要频繁切换选项卡。

### 设置项割裂

在旧版界面里，一些抓取设置分散在“抓取”和“更多”选项卡里。一些命名、下载设置也分散在“下载”和“更多”选项卡里。同一分类的设置分散在两个选项卡里导致设置割裂。

在新版界面里，同一分类的设置会集合在一起，不会割裂。

### 一级分类导航不够方便

在旧版界面里，大部分设置都放在了“更多”选项卡里，导致“更多”选项卡里保存而了全部的 5 个一级分类和数十个设置，但没有提供便捷的分类跳转功能，这导致用户很难快速定位到自己需要的一级分类。

在新版界面里，全部的 5 种设置分类都会显示在左侧，用户可以直接点击分类名字查看该分类下的所有设置。

### 没有二级分类

大部分一级分类都有超过 10 个设置，有些甚至达到了 30 个设置。

旧版界面里这些设置没有二级分类，太多的设置挨在一起，导致查找特定设置的效率很低。新版界面从一开始就设计了二级分组，方便用户快速定位到需要的设置。

### 没有搜索功能

下载器的设置数量已经接近上百个，旧版界面没有搜索功能，用户必须逐个查看设置并判断这是不是自己需要的功能。

新版界面添加了搜索功能，让用户在寻找特定设置时更加便捷。

### 设置项之间难以区分

在旧版界面里，所有设置项都是纯文字，从上到下排列。这导致在大段文字之中很难区分每一条设置的边界。

新版界面里每个设置都显示为独立的卡片，天然具有分界线。

### 不够美观

旧版界面基本上就是单纯的文字和按钮的混排，缺少美感和设计感，有些用户觉得界面复古。

新版界面以 Clash Verge 的界面布局、卡片设计为灵感，使界面的美观程度得到了很大提升。

## 重构时的约束

### 配色方案

标题栏的背景颜色是白色 `#fff`，导航栏的背景颜色是浅灰色 `#f5f5f5`，内容区域的背景颜色是稍微深一些的 `#eee`。

主题色是 `#199df5`。

图标按钮、输入框在 hover 和点击时，如无特别说明，那么它们的高亮颜色就是主题色。

内容区域的文字默认颜色是 `#444`。有些元素会设置更具体的颜色来覆盖这个默认值。

#### 下载器的颜色主题

另外，下载器在不同的颜色主题下，会覆写一些设置的样式。可以参考 `src\style\centerPanel.less`，里面的 `.centerWrap` 有 4 种状态：
- 单独的 `.centerWrap`，这是默认样式，其实也是亮色主题的样式，因为下载器最初只有亮色主题。你为新的设置面板设置样式时，主要是添加在这个选择器里面。
- `.centerWrap.theme-white`，这是亮色主题里的 patch，在默认样式的基础上单独调整一些样式。不过它里面覆写的样式不多，因为下载器的默认样式就是亮色主题。 
- `.centerWrap.theme-dark`，这是暗色主题里的 patch，在默认样式的基础上单独调整一些样式。
- `.centerWrap.xzBG`，这是用户启用了“背景图片”设置时的 patch（会在设置面板下方显示一张背景图片）。

在实际使用中，`.centerWrap` 一定会有 `.theme-white` 或 `.theme-dark` 其中的一个。
`.xzBG` 则是当用户启用了“背景图片“设置之后才会添加。它可以与颜色主题共存，并且实际上它相当于是一种特殊的颜色主题，会覆盖很多样式。

目前的设计稿是在亮色主题下的表现，没有制作其他主题下的设计稿。你可以参考已有的样式来设置一些元素在其他主题下的样式。通常来说，在除了亮色主题之外的主题里，背景颜色比较暗，所以设置面板上的文字会显示为浅灰色或白色。一些元素的背景颜色可能变成透明的或者半透明。一些投影的颜色也会从蓝色变成很淡的白色。

### 样式

所有元素都是 `box-sizing: border-box;` 盒模型。

内容区域里的正文文本字号是 `14px`。一级导航、分类标题等文字需要适当使用更大的字号，例如一级分类的名字的字号使用 `19px`，卡片的标题使用 `16px`。

文字默认都不加粗，除非特别说明需要加粗的。

### 内容和功能

设置面板的容器元素保持已有的 className。最外层元素依然是 `src\ts\CenterPanel.ts` 里的 `<div class="centerWrap ${'lang_' + lang.type}">`，因为有些功能依赖 `centerWrap` 这个类名。主体区域容器的 className 里要有 `centerWrap_con` 以保持样式的兼容性。

一些大的板块元素可能有滚动条（例如导航栏、内容区域），需要给它们添加 `beautify_scrollbar` 的 className 以显示美化的滚动条。

设置项的功能需要保持可用，不要因为重构而导致功能集体失效。重点是要为主体区域嵌套一层 `<form class="settingForm">` 元素，因为下载器内部通常是通过 form.key 来读写设置值的。

在内容区域里，大部分元素都是旧设置面板里的元素，只是放到了卡片里，以及或者修改了所属的分类。所以在重构时，对于卡片里的内容，优先复用已有的代码、样式和逻辑。这样很多逻辑（例如下载区域何时显示、其内部按钮的显示与隐藏）都不需要重新进行处理了。

每个设置的卡片元素需要和之前一样设置 `data-no="<编号>"` 属性，因为一些模块会使用这个数据属性来查找该设置的元素（卡片），对其执行隐藏、显示、移动节点等操作。

### 文本

有时我会明确说明一些按钮的 title 属性，或者一些文字内容。这些文本不能直接设置成固定的字符，需要使用多语言文本配置（可以参考旧代码中的例子）。

**补充说明：**
- 每个一级分类名称前面都会显示一个图标。每个名称都有两套图标文件：未激活时显示的是轮廓较细的图标（line 风格，视觉效果比较轻），激活后显示较粗的图标（fill 风格，视觉效果比较重）。
- 点击一级分类就会激活它，使它高亮显示，并在右侧显示该分类里的内容。
- 默认激活的分类是“首页”，它的作用是仪表盘。
- 同时只能激活一个一级分类。
- 需要记录上一次激活的分类、当前激活的分类，在有需要时使用。当搜索页面隐藏之后（因为用户清除了搜索内容），需要自动切换到上一次激活的分类。至于当前激活的分类，可能会在一些代码逻辑里使用。
- 每当激活左侧的一级分类时，隐藏其他分类的右侧区域，并显示当前分类的右侧区域。还需要检查它里面应该显示哪些设置项。如果某个设置项应该显示在这里但却没有显示（因为它可能被移动到了搜索结果里），就把这个设置项移动过来。
- “搜索”分类默认是隐藏的，所以用户不能主动激活它。只有当需要显示搜索结果时才会激活搜索分类，并在右侧区域显示搜索结果。

每个一级分类在未激活时没有背景颜色，图标、名字的颜色都是 `#111`。

#### 下载摘要区域

在第一个板块里的导航区域后面有一个独立的区域作为下载摘要栏。

下载摘要区域是默认隐藏的，只有当有抓取结果之后才会显示（即触发了这两个事件之一 `[EVT.list.crawlComplete, EVT.list.resume]`）。

下载摘要栏里有两个板块：
- 第一排显示下载状态图标和下载总进度。下载状态有 5 种状态，分别使用不同的图标。这些状态分别是：未开始下载、下载中、下载已暂停、下载已停止、下载完毕。这一排的图标和文字都是灰色的`#666`。

## 卡片的设计

### 卡片的视觉效果要轻

卡片要轻。因为设置数量很多，所以每个卡片的视觉效果不应该太重。现在卡片的样式是：
  - 背景是白色 `#fff`
  - 没有边框
  - 卡片具有较轻的投影。一级卡片的投影是 `0 0 15px #ddd`，二级卡片的投影范围小一些，是  `0 0 10px #ddd`。
  - 主要靠留白和分组来区分，每个卡片之间有一定间距。

### 卡片的种类

下面的每种卡片都带有示意图。这些示意图不是精确的设计稿，仅供参考。

每种卡片的宽度都是自适应的。默认会占满可用宽度，但也可以通过设置样式使其只占据部分宽度。

#### 标题卡片

用于显示二级分类的名字。

左侧是图标和分类名字。图标有可能有，也可能没有。没有图标的话就不显示图标的元素。
右侧有一个箭头图标表示展开、折叠状态。其实它只是同一个图标，箭头是向下的。当这个分类折叠时，正常显示这个图标（旋转角度为 0）；当这个分类展开时，把这个图标旋转 180 度，使箭头向上。

事件：
点击标题卡片会切换其下内容区域的展开/折叠状态。不需要单独为箭头图标绑定事件，因为点击整个标题卡片都可以展开/折叠。

##### 样式

它总是一级卡片，具有 `margin-top: 15px;` 和 `padding: 8px 15px;`，总高度为 `50px`。背景颜色是白色，圆角 `8px`。

分类名字和箭头是左右两端布局。其中的元素在水平轴上居中对齐。

图标容器的宽高是 `32px`，背景颜色 `#eef7fd`，圆角 `6px`。内部的图标宽高为 `20px`，图标颜色是主题色，居中显示。

右侧箭头图标的文件是 `arrow-down-2.svg`，宽高为 `16px`。

#### 内容卡片

用于直接显示内容的卡片。没有标题、箭头等结构，是很简单的卡片。它被用来显示多种内容，例如提示信息、设置项、帮助分类里的按钮。

内容卡片可能是一级卡片，也可能是二级卡片：
- 如果它不隶属于二级分类，那么就是一级卡片，例如首页顶部的帮助信息、帮助页面里的小卡片。
- 当它用于显示设置项时，如果它隶属于二级分类，那么它是二级卡片，它的宽度会比标题卡片稍微窄一些，以表示层级关系。这应该可以通过在父元素上设置样式来实现。
- 当它用于显示设置项时，该设置可能具有置顶属性、new 属性。这两种属性有不同的表现形式，具体可以查看该文档底部的“置顶设置功能”和“新增设置的表现形式”章节。

##### 样式

如果它是一级卡片（没有显示在标题卡片下）具有 `margin-top: 15px;`，如果是二级卡片则具有 `margin-top: 10px;`。它的 padding 总是 `padding: 12px 15px;`。背景颜色是白色，圆角 `12px`。

如果它是二级卡片，那么宽度比一级卡片少 `20px`，就是左右两侧各少 `10px`，并且保持居中。

字号是 `14px`，行高 `26px`。

由于内容卡片里主要显示设置项，并且设置项的 html 代码没有大的改变，所以可以优先复用之前的样式。

#### 标题+内容卡片

相当于一个独立的板块，上方是标题栏，下方的内容区域里可以放置任意内容。

标题栏：内容与“标题卡片”相同，左侧是图标和名字，右侧有一个箭头图标表示展开、折叠状态。

目前我打算用它来显示首页仪表盘里的开始抓取、附加功能、下载管理区域。

## 搜索功能

### 搜索匹配范围

- 从所有设置项的配置数据里查找（而不是从 DOM 元素里查找）。
- 优先从每个设置项的名字（`name` 属性）里匹配搜索词。如果名字有匹配结果，就不再查找它的搜索词列表 `searchWords`。当这个设置项的名字没有匹配的结果时才会在它的搜索词列表里查找。
- 从搜索词匹配时，依次使用两种匹配模式：第一种是 `searchWord.includes(inputWord)`，这适合用于“搜索词的文本覆盖了用户输入的文本”的情况。但有些关键词比较短，如果用户输入的搜索词比较长，`searchWord.includes` 就匹配不到，例如用户输入了 `修改语言`，但搜索词里只有更短的 `语言`。所以如果第一种匹配模式没有结果，就使用第二种匹配模式 `inputWord.includes(searchWord)` 反过来匹配一次。
- 由于搜索词列表 `searchWords` 是数组，所以在从某个设置里进行搜索时，一旦有匹配的搜索词，就不再匹配剩余的搜索词，因为没有必要了。
- 不查找设置的子选项。因为设置项的名称与搜索词的关联度是最高的，子选项则不够高。有些子选项会出现在多个设置里，但这些设置的作用差别很大，所以没有太大必要对子选项进行搜索。这样也可以减少工作量，例如：1. 如果要搜索子选项的话，就需要在配置数据里添加子选项的文本，而且数量很多。2. 有些设置是未启用的，如果搜索子选项的话，是否要搜索未启用的设置的子选项？这是个选择。如果进行搜索，还会面临进一步的问题：对于未启用的设置，其子选项区域往往是隐藏的，如果在隐藏的子选项里找到了搜索结果，要让用户看到这些子选项，就需要临时把子选项显示出来。这也是个麻烦的地方，所以不处理。
- 不查找“帮助”里的内容，因为它里面的内容不属于设置。

### 如何呈现搜索结果

- 用户输入搜索词时，自动查找并显示搜索结果。为了避免短期内多次计算的压力，需要使用防抖处理（等用户停止输入 0.2 秒之后才开始查找搜索结果）。
- 不在搜索框下面显示下拉的候选项列表，因为这样会增加复杂度。而且目标设置本身就可以在设置面板里显示，所以没太大必要显示下拉候选项。
- 当用户在搜索框里输入内容之后（搜索词非空），就激活“搜索”分类，并在其右侧内容区域（全局搜索页面）里显示符合条件的设置。
- 搜索页面的顶部会显示有多少条搜索结果。如果没有符合条件的搜索结果，显示“没有找到符合条件的设置。你可以尝试搜索更简短的关键词。”。如果有搜索结果，则显示`找到 x 条与“搜索词”有关的设置`。这个区域的样式：`margin:15px 0 0 5px`，文字从左到右显示，颜色是 `#444`。
- 把搜索到的设置项从它们的二级分类区域里移动到搜索结果区域里。这些设置项可以直接使用，所以点击它们时，不需要跳转到它们原本所属的一级分类里。
- 在搜索到的设置里，如果设置的标题里有命中的文字，则高亮显示命中的文字，使其具有黄色背景颜色。目前使用黄色 `#ffff00`，如果以后觉得太亮的话可以修改成更柔和的颜色。
- 搜索结果可能会来自多个一级分类，在呈现搜索结果时，这些设置的显示顺序应该与左侧一级导航一致：依次是：首页 → 抓取 → 命名 → 下载 → 增强 → 其他。
- 在每个一级分类内部，再按原本的二级分组顺序显示。也就是以“一级分类”+“二级分类”为搜索结果进行分组，例如 `下载 / 小说`，并使用标题卡片来显示这个组合的分类名称。

何时隐藏搜索页面：
未搜索或搜索词为空时，隐藏搜索分类页面，并重新激活上一次激活的一级分类。搜索词为空的情况可能由多种触发方式：
- 当用户点击搜索框右侧的 x 图标时会清除搜索词
- 当用户手动清空了搜索词时（例如连续按 backspace 键）。

可以通过监控搜索框 input 控件的 value 来检测搜索词是否为空。

被置顶的设置会集中显示在“首页”分类的内容区域里。

#### 置顶图标的外观

被置顶的元素以前会在其左侧显示一个置顶图标（📌），现在改为在其卡片**内部**显示边框，粗 `2px`，颜色是 `#bde2ff`。这个边框具有与内容卡片一致的圆角。这样被置顶的设置设置会变得更加醒目，界面整体上也更加美观。

#### 怎么置顶和取消置顶

在设置项的卡片上长按一段时间（例如 0.5 秒），就可以切换该设置的置顶状态。旧版的代码里有对应的逻辑，可以参考。

#### 被置顶的设置只有一份元素

当左侧导航区域激活了“首页”分类时，查找所有被置顶的设置，把它们的元素从其原本所属的分类区域里移动到“首页”的置顶设置区域里。

当左侧导航区域激活了非“首页”分类时，该分类里被置顶的设置会正常显示在其原本的位置，依然附带边框。也就是说，置顶的设置可以在两个地方看到：一是首页的置顶分类区域，二是其原本的位置。

由于被置顶的设置在初始情况下会被移动到“首页”里，所以当用户激活其他分类时，需要把里面的置顶设置从“首页”里移动到当前分类区域里。这样，用户在其他分类里依然能看到被置顶的设置，不会产生“这个设置怎么不见了”的困惑。

这样做的目的是为了让置顶的设置只有一份元素，而非多份。原因是下载器在设计时没有考虑到一个设置会存在多份的情况。如果一个设置存在多份，会带来问题：
- 由于设置项都包裹在一个 form 表单里，并且使用 form.key 的方式来读取、设置对应的值。如果被置顶的设置存在多份，那么同一个 key 会出现多次，造成问题。
- 一些模块需要查找特定的设置元素并对其进行操作（如读取、写入一些控件的值，或者操作里面的 DOM）。如果有多个相同的设置，就需要遍历这些重复的设置来挨个处理，并且同步数据和元素，会比较麻烦。

## 新增设置的表现形式

新增设置的处理逻辑在 `src\ts\setting\ShowNewIcon.ts` 里，现在只需要修改表现形式。

之前新设置会在它的左侧显示一个 ✨ 符号，现在改为在卡片的右上角显示一个 new 徽章，对应的图标是 `new.svg`。

## 独立的提示消息卡片

设置面板里有一些重要的提示，使用单独的元素显示，占据一行。

在旧版界面里，就是 `src\ts\CenterPanel.ts` 里的 `p id="tipPinOption"` 元素。它的内容由 3 部分组成：
1. 前面的灯泡图标 💡
2. 提示内容的文本
3. “我知道了”文字按钮，点击之后不再显示这条提示。

现在这些重要提示改为使用独立的内容卡片来显示。在“帮助”页面的设计稿里，顶部的 3 个卡片就是重要提示：

![](./设计图-帮助.png)

每个提示的内容依然是 3 部分，但发生了一些变化：
1. 灯泡图标💡从 emoji 符号变成了 svg 图标（图标文件是 `light-line.svg`）。图标是灰色的 `#666`。
2. 提示内容没有变化，依然是纯文本
3. “我知道了”按钮从文字变成了一个对号 svg 图标（图标文件是 `yes_submit.svg`），title 为“已确认”。

这个对号图标的样式：
- 宽 `30px`，高 `20px`，颜色是主题色，圆角 `4px`。
- 当鼠标 hover 该图标时，显示背景颜色 `#e5f2ff`。

灯泡图标和对号图标都需要与文字行垂直居中对齐。
