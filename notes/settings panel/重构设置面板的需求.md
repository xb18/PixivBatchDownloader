# 重构设置面板

日期：2026-05-17

## 设置面板的功能介绍

这个项目是一个 Pixiv.net 下载器。由于它是浏览器扩展程序，运行在 Pixiv.net 的网页里，所以它的界面里不需要实现浏览 Pixiv 上的图片、小说的功能，只有与下载相关的功能。

主要有 3 种功能：
- 显示设置项
- 显示一些功能按钮，例如开始抓取、开始下载、或者其他一些功能。
- 显示和管理下载进度。

在下面的内容里，“设置”和“选项”都用来表示设置面板上的具体设置项。它们的含义是相同的。

## 目前的界面设计和截图

设置面板的主要内容是 3 个选项卡，它们可以切换，同时只会显示一个。

第一个选项卡标题是“Crawl”，保存了一些相关设置。截图文件是 `\notes\settings panel\tab-1.png`。

第二个选项卡标题是“Download”，保存了命名、下载相关的的设置。当开始下载时，也会在这里显示下载进度区域。截图文件是 `\notes\settings panel\tab-2.png`。

第三个选项卡标题是“More”，截图文件是 `\notes\settings panel\tab-3.png`。第三个选项卡里有 5 个子分类，截图里只显示了第一屏，没有显示完。所有分类如下：
- Crawl：保存了一些抓取设置。其实第一个选项卡里的设置也属于此分类，但是为了让用户能更方便的使用常用的抓取设置，所以我把一些抓取设置放到了第一个选项卡里。
- Naming：保存了一些命名设置。其实第二个选项卡里有一些设置也属于此分类，为了让用户更方便的使用常用的命名设置，我把一些设置放到了第二个选项卡里。
- Download：保存了一些下载设置。其实第二个选项卡里有一些设置也属于此分类，为了让用户更方便的使用常用的下载设置，我把一些设置放到了第二个选项卡里。
- Enhance：保存了一些增强功能的设置。
- Others：保存了一些不属于上面分类的杂项设置。

## 设置面板的 html 代码

设置面板的 html 代码在源文件里是分布在多个文件里的，最终组织成完整的设置面板：
- `src\ts\CenterPanel.ts` 的 `addCenterPanel` 方法里保存了设置面板的容器和外围元素。其中有个插槽元素 `<slot data-name="form"></slot>`。
- `src\ts\setting\Form.ts` 会在上面的插槽元素里插入来自 `src\ts\setting\FormHTML.ts` 的 html 代码 `formHtml`。`formHtml` 里也有一些插槽，其他多个模块都会向特定的插槽里添加元素。这些具体的模块不是重点，你只要知道有多个插槽即可。重构时需要保留这些插槽。

由于设置面板的最终 html 代码是动态生成并插入到 pixiv 的网页里的，所以该项目里看不到最终的 html 代码。作为代替，我从网页上拷贝了设置面板的完整代码，保存到了 `\notes\settings panel\html and style.html` 里。这个 html 文件的 body 标签里的代码是最终 html 代码，如果你有需要的话可以参考。

由于这个 html 文件里没有添加 js 代码，所以只有静态 html 内容，无法切换选项卡，也无法触发事件。

这个 html 文件里使用的 css 文件是 `\notes\settings panel\style.css`，其实就是 `dist\style\style.css`。

## 选项的显示与隐藏、以及置顶选项功能

对于设置项，有一些额外处理：
- 有一个“显示高级设置”（“Show advanced settings”）开关。它默认是启用的，此时只有少数设置会显示出来（它们是始终显示的），大部分设置都会隐藏。这是为了避免用户看到太多不必要的设置。把它禁用的话则会显示所有选项。
- 有一个功能是把选项置顶，被置顶的选项会显示在选项卡的顶端，并始终显示（不管是否启用了“显示高级设置”）。

## 效率优先

目前的设计原则是效率优先。

- 为了提高使用效率，选项都是直接显示的。
- 为了避免太多设置让用户无从下手，或者降低查找设置的效率，所以我默认启用了“显示高级设置”功能，让大部分设置默认隐藏。
- 分类：所有选项都根据功能进行了分类。分类可以提高用户查找设置的效率。所有选项都可以归为“More”选项卡里的 5 种分类。同时，为了让用户可以更直接的看到常用的、重要的设置，我把一些设置单独放在了第一个和第二个选项卡里。

## 目前存在的问题

- 相邻设置难以区分：选项是从上到下排列的，而且这些选项都是纯文字，导致相邻设置难以区分。虽然我在每个设置的名字里都高亮了一些关键字，可以在不同设置之间做出区分，但是当同时显示了太多设置时，仍然难以区分它们。每个选项之间缺乏视觉上的区别。
- 一些分类割裂：第三个选项卡“More”里有二级分类，但其中一些分类的部分设置被放到了第一个或第二个选项卡里，没有统一到同一个分类里。我的一个设想是把每种分类严格的放到一起，这样只有 5 种分类，进一步，可以做成 5 个选项卡来取代现在的 3 个选项卡。但是这样做也有一些问题：如果把同一个分类的设置都放在一起，分类内内部的设置数量太多了，会导致用户难以从中找到常用的设置。
- 第三个选项卡“更多”里的设置数量太多，太重了。按照里面的二级分类进行拆分会好些。
- 不够美观：选项数量太多了（差不多有 100 个），纯文字很多，看起来不够美观，不够现代化。但是如果添加其他美化元素（包括使用背景颜色）的话，需要注意不能让可读性变差的太多。

## 我想让你帮我做什么

在保留所有选项、按钮、下载区域的前提下，思考**如何重新组织这些元素的显示方式**，目的是提高用户体验，并解决或缓解目前存在的一些问题。

**你可以对分类、布局做出大胆的修改**，也可以使用全新的布局方式。

你不需要刻意保持目前的 3 个选项卡的布局方式，就像上面我设想过的，可以改成 5 个选项卡；也可以考虑把选项卡改成在左侧纵向排列的（而不是目前的横向排列）；也可以使用其他你认为比较好的方案。甚至不使用选项卡也是可以的。

分类和布局需要有合理性，同时不要让使用效率降低太多。对于上面说的“不够美观”的问题，可以考虑做一些美化，但这不是重点，如果在美化方面没有好的方案，可以先不考虑美化。
